
자바의 스레드는 특정 시점에 다음 6가지 상태 중 하나에 속해 있다.

- NEW
	- `new Thread(r)`을 통해 스레드 객체를 방금 생성했지만, 아직 `start()`메서드를 호출하지는 않은 상태이다. 
- RUNNABLE
	- `start()` 메서드가 호출된 직후의 상태
	- 실제로 코드가 cpu에서 실행되고 있을 수도 있고, 아니면 OS가 cpu를 할당해 주기를 기다리는 상태일 수 있다. 자바는 이 둘을 구분하지 않고 `RUNNABLE`이라 부른다.
- BLOCKED
	- 다른 스레드가 사용중인 고유 락(Intrinsic Lock)을 얻기 위해 뭘 하진 않고 기다리는 상태이다.
- WAITING
	- 다른 스레드가 어떤 특정한 행동을 해주기를 기다리는 상태이다.
	- 예를 들어, `Thread.join()`을 호출하면 대상 스레드의 작업이 완전히 끝날 때까지 이 스레드는 `WAITING`상태가 된다.
- TIMED_WATITING
	- `WAITING`과 비슷하지만 기다리는 시간이 정해져 있는 상태이다.
- TERMINATED
	- 스레드의 활동이 끝난 상태
	- `run()`메서드 안의 코드를 모두 실행하여 자연스럽게 종료된 상태거나, 예외가 발생하여 종료된 경우이다.

주요 메서드
- `Thread.yield()` : 현재 실행 중인 스레드가 남은 cpu 시간을 다른 스레드한테 양보
- `t.join()`/ `t.join(long millis)` : 대상 스레드(`t`)가 끝날 때까지 기다림. 시간을 주면 그 시간동안만 기다린다.

쓰면 안되는 메서드
- `stop()` : 스레드를 즉시 죽이는 메서드로 이 방식을 쓰면 스레드가 하던 작업을 제대로 마무리하지 못하고 데이터가 훼손될 수 있다.
- `suspend()`, `resume()` : 스레드를 강제로 일시 정지, 재개시킨다. 이 메서드들은 deadlock을 유발하기 쉽다.
- 위 세 메서드는 현재 폐기처분됨...
