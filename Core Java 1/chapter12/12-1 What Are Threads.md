자바에서 멀티스레딩을 구현하기 위해서 알아야 할 두가지 요소가 있다. `java.lang.Runnable` 인터페이스, `java.lang.Thread` 클래스이다.

`Runnable` 인터페이스
- 스레드가 실행해야 할 작업을 정의하는 인터페이스
- 내부에는 `void run()`이라는 단 하나의 메서드만 존재한다. 이 `run()` 메서드 안에 스레드를 통해 실행하고픈 코드를 작성하면 된다. 
- 함수형 인터페이스이므로 람다식으로 표현할 수 있다.
`Thread` 클래스
- 앞서 만든 작업을 스레드에서 실제로 돌리는 역할

이 두 요소를 결합하여 별도의 스레드에서 작업을 실행하는 순서는 다음과 같다.
1. Runnable 인터페이스를 구현하여 `run`메서드 안에 실행할 코드를 넣는다.
```java
Runnable r = () -> {...}
```
2. Runnable 객체 r을 Thread 클래스의 생성자에 전달하여 스레드 객체를 만든다.
```java
Thread t = new Thread(r);
```
3. Thread 객체의 start() 메서드를 호출한다.
```java
t.start();
```

주의사항
1. `t.run()`을 직접 호출하면 안된다. 만약 직접 호출하게 되면 새로운 스레드가 생성되는 것이 아니라 메인스레드에서 메서드 호출이 되버린다. `start()`메서드를 호출해야 내부적으로 OS에 접근하여 새로운 스레드를 생성한다.
2. 과거에는 `class MyThread extends Thread` 처럼 Thread 자체를 상속받아 `run()`을 오버라이딩 하는 방식도 썼다. 하지만 교재는 이 방식을 권장하지 않는다. 나중에 다룰 Thread pool 환경에서는 작업이 매우 많아진다, 매 작업마다 `Thread` 객체를 새로 찍어내는 것은 오버헤드가 크다. 그렇기에 작업 내용만 `Runnable`로 만들어서 넘기는 방식이 현재 자바 프로그래밍의 올바른 규범이라 할 수 있다.

추가 개념
- `Thread.sleep(long millis)` : 현재 실행 중인 스레드를 인자로 전달된 밀리초 시간동안 잠시 재움
- `InterruptedException` : 자고 있는 스레드를 다른 스레드가 강제로 깨우거나 종료하려 할 때 발생하는 예외. 교재에서는 스레드를 안전하게 종료시키기 위한 신호로 이 예외를 사용한다고 한다. 따라서 sleep을 사용할때는 try-catch 블록으로 이 예외를 처리해주어야 한다.
