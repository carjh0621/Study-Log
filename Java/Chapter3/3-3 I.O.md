
## input

The Scanner class is defined in the `java.util` package.
	`import java.util.*;`
- `Scanner(InputStream in)`  
	constructs a Scanner object from the given input stream.  
	`Scanner in = new Scanner(System.in);`
- `String nextLine()` 
	reads the next line of input. (contain spaces)  
	`String name = in.nextLine();`
- `String next()`  
	reads the next word of input (delimited by whitespace).  
	`String firstName = in.next();`
-  `int nextInt()`  
- `double nextDouble()`  
	reads and converts the next character sequence that represents an integer or floating-point number.  
- `boolean hasNext()`  
	tests whether there is another word in the input.  
- `boolean hasNextInt()`  
- `boolean hasNextDouble()`  
	tests whether the next character sequence represents an integer or floating-point number.

## file i/o

읽기

```java
import java.io.IOException;  
import java.nio.charset.StandardCharsets;  
import java.util.*;  
import java.nio.file.Path;  
public class FileIO {  
    public static void main(String[] args) throws IOException {  
        Scanner in = new Scanner(Path.of("myfile.txt"),  
                StandardCharsets.UTF_8);  
    }  
}
```

If the file name contains backslashes, remember to escape each of them with an additional backslash: `"c:\\mydirectory\\myfile.txt".
주의
``` java
Scanner in = new Scanner("myfile.txt");
//- `"myfile.txt"`를 **파일명이 아니라 데이터로 인식**
//문자 `'m'`, `'y'`, `'f'` … 를 읽는 Scanner 생성
//파일을 읽고 싶다면 반드시 `Path` 또는 `File`을 써야 함
```

쓰기
```java
PrintWriter out = new PrintWriter(
    "myfile.txt",
    StandardCharsets.UTF_8 // 이 파일을 UTF-8로 해석하겠다
);
//파일 쓰기 
- 파일이 없으면 → **새로 생성**
- 파일이 있으면 → **덮어씀**
```
`PrintWriter`는 내부적으로 **버퍼링**을 함.  프로그램이 끝나기 전에 닫히지 않으면:
	파일이 비어 있거나 일부만 써질 수 있음
-> `out.close();`
-> try-with-resources (자동으로 `close()` 호출)
```java
try (PrintWriter out = new PrintWriter(
        "myout.txt", StandardCharsets.UTF_8)) {

    out.println("Hello");
    int x = 3, y = 7;
    out.printf("%d %d%n", x, y);
}
```
- `%n` → **플랫폼 독립 줄바꿈**
- Windows / Linux / macOS 차이 흡수

예외
- 읽으려는 파일이 없음
- 쓸 파일을 생성할 수 없음
- 권한 문제
- 디스크 I/O 오류
-> `throws IOException` : “입출력 오류가 날 수 있다는 걸 알고 있으며, 여기서는 직접 처리하지 않고 JVM에 맡기겠다”

읽기쓰기
**읽고 쓰려면 Scanner와 PrintWriter 둘 다 필요하냐?**
- 개념적으로는 **읽기 객체 + 쓰기 객체가 각각 필요**
-  반드시 이 둘일 필요는 없지만, 가장 직관적인 조합
다만, - `PrintWriter` 생성 시점에 **파일을 바로 비움** -> 읽기도 전에 데이터가 날아갈 수 있음
