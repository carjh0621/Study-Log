
## 1. 배열과 초기값

**참조형 배열 (객체 배열)** : `Employee` , `String` 등 클래스 타입의 경우.
- 초기값 : `null`
**`Primitive 배열`** : `int`, `double` , `boolean` 등
- 초기값: 0, 0.0, `false`

ex) `Employee[][] e = new Employee[n][m];`
- n x m 공간만 힙에 만들어진 상태
- 객체는 아직 생성되지 않은 상태
- 초기값: `e[0][0]` 부터 `e[n-1][m-1]` 까지 전부 `null`
- 전체 돌면서 하나씩 `new` 해줘야함

## `String`의 경우

1. 리터럴 방식
```java
String a = "Hello";
String b = "Hello";
```
힙 안에 있는 **String Constant Pool (문자열 상수 풀)** 이라는 공간을 사용함
- `a` 가 "Hello" 를 풀에 만듦.
- `b`가 풀에 이미 있는 "Hello" 의 주소를 가리킴.

2. new 사용
```java
String c = new String("Hello");
String d = new String("Hello");
```
풀을 거치지 않고, 힙 영역에 새로운 객체를 만듦.
- `c` 와 `d` 는 서로 다른 주소를 가짐

3. 사용자 입력
```java
String input = scanner.next(); // Hello 입력
```
사용자 입력으로 받은 문자열은 리터럴이 아니므로, 런타임에 힙 영역에 새로운 객체로 생성됨.
- `new`를 쓴 것과 비슷하게 동작함
