
Abstract Class
- 상속 계층 구조의 위쪽으로 갈수록 클래스는 점점 일반적이게 된다.
- 예를들어, Employee 와 Student의 공통 조상으로 Person을 만든다고 하자.
- 그렇다면 사람이라는 개념은 존재하지만, 현실 세계에서 사람이라는 객체를 만드는 것은 이상하다. (직업이나 신체 수치, 주민번호등의 특성으로 대표하는게 더 적절할 것 )
- `Person`을 추상 클래스로 선언 --> 단지 설계도임을 명시 (`new`로 생성하지 말 것을 명시)

Abstract method
- 선언부만 있고, {}는 없는 메서드. 
- ex: `public abstract String getDescription();`
- 자식 클래스들이 알아서 기능 구현

규칙
- 추상 메서드가 하나라도 있으면, 그 클래스는 무조건 추상 클래스여야 한다.
- 추상 클래스는 인스턴스 생성 불가 
- 변수 타입으로는 사용 가능 (다형성): `Person p = new Student();`
- 자식 클래스는 상속받은 추상 메서드를 빠짐없이 모두 오버라이딩 해야한다. 
	- 만약 하나라도 구현하지 않으면 자식도 추상 클래스가 되어야 한다.

쓰는 이유
- `name` 같은 공통 필드와 기능은 부모에 두고 재사용
- 모든 클래스에게 특정 기능을 강요할 수 있음
	- ex: `getDescription()` 같은 메서드를 추상 메서드로 두면, 모든 자식 클래스가 강제로 본인만의 소개 멘트를 구현해야함.
